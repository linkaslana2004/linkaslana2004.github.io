java面向对象编程

2024-10-27 2024-10-31

java面向对象的定义

  • 面向对象的思想
    • 使用物理类聚的方法，即分类思维模式，思考问题首先会解决问题需要哪些分类，然后对这些分类进行单独思考。最后，才对某个分类下的细节进行面向过程的思考。
    • 面向对象适合处理复杂的问题，适合处理需要多人协作的问题。
  • 对于描述复杂的事物，为了从宏观上把控，从整体上合理分析，我们需要使用面向对象的思路来分析整个系统。但是，具体到微观的操作，仍然需要面向过程的思路去处理。
  • 面向对象编程（OOP）本质是：以类的方式组织代码，以对象的组织（封装）数据。

面向对象编程的四大特性

  • 封装Encapsulation
    将数据（属性）和操作数据的方法封装在对象内部，使用访问控制来限制外部访问。
  • 继承（Inheritance）：
    子类继承父类的属性和方法，并可以进行扩展。
  • 多态（Polymorphism）：
    同一操作在不同对象上表现出不同的行为。
  • 抽象（Abstraction）：
    通过抽象类或接口隐藏复杂的实现细节，暴露简单的接口。

类与对象

  • 类是一种抽象的数据类型,而对象则是具体概念的具体实例
    • 类是一个模板或蓝图，定义了某种事物的属性（变量）和行为（方法）。它本身不占用内存，只是描述了对象该如何构建。
    • 对象是类的实例。也就是说，通过类创建出来的具体个体就是对象。每个对象都有自己独立的数据。
    • 举一个例子就是游戏是一个类，而黑神话：悟空则是一个对象。

创建对象：

  • 使用new的关键词进行对象的创建。需要注意的是，除了分配内存空间之外，还会给创建好的对象，进行默认的初始化以及对类中构造器的调用。
  • 类中的构造器也称为构造方法，在类的对象创建时自动调用。它的主要作用是为新创建的对象赋初始值。
    • 构造器是由类创建出来的每个对象都要有的属性
    • 构造器的名称必须和类的名字相同
    • 必须没有返回类型，也不能写 void
    • 在创建对象时自动调用
    • 可以有参数
    • 可以重载，实现多种初始化方式
  • 类中的成员方法是在类中构建出的，用于定义对象行为的方法，其可由对象调用来表示对象所拥有的某种属性。
    • 其通常包括访问修饰符public、private、protected；返回类型；方法名；参数列表以及方法体

封装

  • 封装是是将数据（属性）和操作数据的方法封装在对象内部，并通过访问控制来限制外部对数据的访问和修改。

封装的好处

  • 可以保护数据的完整性
    • 避免外部程序直接修改对象的状态，防止非法数据的产生。
  • 隐藏内部实现细节
    • 只暴露必要的接口，使得代码更加简洁、可维护。
  • 易于维护和扩展
    • 通过方法访问数据，便于在不影响外部代码的情况下进行修改和升级。

封装的实现步骤

  • 将类的属性设置为 private，使得它们不能被类外部直接访问。
  • 通过public的getter和setter方法提供对属性的访问接口
  • 在setter方法中进行数据校验，保证数据的合法性。

封装的关键点解析

  • private访问修饰符：
    • 将属性声明为private，确保它们只能在类内部访问。
  • getter和setter方法：
    • getter用于获取属性值，setter用于修改属性值。
    • 在setter方法中可以进行数据校验，防止不合理的值被赋给属性。
  • Java 中的访问修饰符决定了类的属性和方法的访问范围
    • private：只能在类内部访问。
    • public：可以被类外部的所有代码访问。
    • protected：在同一个包内或继承的子类中访问。
    • default（不写修饰符）：只能在同一个包内访问。

继承

  • 允许一个类继承另一个类的属性和方法。通过继承，子类可以复用父类的代码，并在此基础上进行扩展或修改。

继承的基本概念

  • 父类(Super Class)/基类(Base Class)
    • 提供原始属性和方法的类，即提供给子类继承的类
    • 还有一个别称为超类
  • 子类(Sub Class)/派生类(Derived Class)
    • 继承父类的类
    • 继承了父类的所有非私有成员（包括属性和方法），并且可以添加新的属性和方法，或重写（Override）父类的方法。
  • extends关键字
    • 用于表示继承关系。子类使用 extends 关键字继承父类。

多态

  • 多态（Polymorphism）在 Java 中是面向对象编程的核心特性之一。它的本质是同一接口、不同实现。通过多态，我们可以用父类的引用来指向子类的对象，并在程序运行时根据对象的实际类型来调用合适的方法。
  • 多态让代码更具灵活性和可扩展性。开发者可以在不修改已有代码的情况下添加新的功能或类型。

多态的优点

  • java的多态是实现面向对象编程的重要特性。
  • 它具有的灵活性和可扩展性可以使代码可以同时处理不同类型的对象且无序修改代码。
  • 其父类或者接口定义了规范，具体实现由子类完成，降低了代码的耦合度。
  • 其可以使用统一的方法处理多种类型的对象，减少重复代码。

抽象类与接口

抽象类

什么是抽象类

  • 抽象类是无法直接实例化的类，它用于定义一类事物的共性，并且可以包含：
    • 抽象方法（没有实现的方法）
    • 具体方法（有实现的方法）
    • 成员变量和构造器

抽象类的定义

  • 其包括以下几个元素：
    • abstract关键字：用于修饰类或方法，表明这是抽象的。
    • 抽象方法：没有方法体，必须在子类中实现。
    • 具体方法：抽象类也可以包含有方法体的方法。

抽象类的特点

  • 不能直接实例化。
  • 可以有构造器，但只能在子类的构造过程中调用。
  • 可以包含非抽象方法。
  • 可以包含成员变量。
  • 用于描述一类事物的共性。

接口

什么是接口

  • 接口定义了一组方法规范，它类似于一个完全抽象的类，即所有方法默认是抽象的。接口用于定义不同类之间的行为契约。Java8之后，接口也可以包含一些默认方法和静态方法。

接口的定义

  • 其包括以下几个元素：
    • interface关键字：用于修饰，表明这是接口
    • 抽象方法：注意，无需关键字abstract

接口的特点

  • 不能包含成员变量（只能有常量）
  • 支持多继承：一个类可以实现多个接口。
  • 所有方法默认是public和抽象的（Java 8 开始可以有默认方法和静态方法）。
  • 用于定义行为契约，描述某一类事物的功能。

接口的默认方法和静态方法

  • 默认方法（default）：带有方法体，可以在实现类中使用。
  • 静态方法（static）：可以通过接口直接调用。

抽象类与接口类的区别

特性 抽象类 接口
关键词 abstract interface
是否可以包含具体方法 可以 从 Java 8 开始可以有默认方法和静态方法
是否可以包含成员变量 可以（非 static 成员变量） 只能包含 public static final 常量
继承/实现方式 通过 extends 继承 通过 implements 实现
多继承 不支持（一个类只能继承一个抽象类） 支持多继承（一个类可以实现多个接口）
构造器 可以有构造器 不能有构造器
应用场景 用于描述“是什么”的共性 用于描述“能做什么”的行为

抽象类与接口的选择

  • 使用抽象类的场景：
    • 当需要共享代码，且类之间存在（是某种类型）关系时，适合使用抽象类。
    • 例如：Animal 类作为所有动物的基类，共享 eat() 方法。
  • 使用接口的场景：
    • 当需要定义行为规范，且没有具体实现时，适合使用接口。
    • 接口还适用于多个不相关类需要实现同样的行为时。例如，Flyable接口可以用于描述飞机、鸟等可以飞行的类。
